サイトのディレクトリ構成

```bash
sapporo-redevelopment-log/
├── .github/
│   └── workflows/
│       └── publish.yml  <-- 自動更新用の設定
├── docs/                <-- 記事や画像
│   ├── index.md         <-- トップページ
│   ├── hokkaido-univo/  <-- 各記事のフォルダ
│   │   ├── tadao-ando.md <--記事
│   │   └── images/      <-- 写真フォルダ
│   │       └── photo.jpg
├── mkdocs.yml           <-- サイトの設定ファイル
```

＃publish.yml がやっている4つのステップ
1. 監視（Trigger）
「main ブランチ（メインの保存場所）に新しいファイルが保存（Push）されたら、すぐに動き出しなさい」という指示をしている。

2. 準備（Setup）
GitHubが持っている空っぽのコンピュータ（UbuntuというLinux OS）を1台起動する。
MkDocsはPythonというプログラミング言語で作られているので、「このコンピュータでPythonを使えるようにして」と指示しており、いわば料理をするためにキッチンをセットアップしている状態である。

3. 道具のインストール（Install）
サイトを作るのに必要な道具（mkdocs-material というテーマや、画像拡大プラグイン）をインストールしている。

4. 生成と公開（Build & Deploy）
tadao-ando.md などのテキストファイルを読み込み、それを綺麗な Webページ（HTMLファイル）に変換し、変換したファイルを、裏側にある gh-pages という専用の場所にアップロードしている。

#このファイルがないとどうなる？
もしこのファイルがない場合、記事を更新するたびに以下の作業をしなければならない：
* パソコンで黒い画面（ターミナル）を開く。
* コマンドを打って変換処理をする。
* 出来上がったHTMLファイルをまたアップロードする。
publish.yml は、この面倒な作業を全自動化してくれる、非常に優秀な執事のような存在である。
